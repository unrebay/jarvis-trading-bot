# Static vs Dynamic (Статический vs Динамический подход)

## Определения

**Статика** — математика. Действия ограничены определённой последовательностью, ключевую роль играет надёжность. Неизменные правила.

**Динамика** — переменчивость. Параметры подстраиваются под текущий момент, решения принимаются исходя из уникальности каждой ситуации.

## Динамический подход

**Плюсы:**
- Проста для понимания новичком — видишь события, принимаешь решения
- Свобода действий и самовыражения, независимость от жёстких правил

**Минусы:**
- На практике крайне сложно — требует тысячи часов бектеста и живой торговли
- Тяжело тестировать: каждая ситуация уникальна → нет надёжной статистики
- Логика на бектесте и в живой торговле часто расходится → путаница

## Статический подход

**Плюсы:**
- Простота торговли: чёткие, протестированные правила, неизменные в любых условиях
- Торговля становится почти роботизированным процессом
- Бектест даёт точные данные и прозрачность

**Минусы:**
- Создание оптимального набора инструментов — сложная задача, нужен опыт и ментор
- Отсутствие гибкости: нужно брать трейд по правилам, даже если интуиция против

## Static can be Dynamic (Гибридный подход)

Статический и динамический подходы можно комбинировать через **контекст**:

Пример: контекст определяется по одному инструменту — LG PDL/PDH:
- Снимается PDL → контекст лонговый
- Снимается PDH → контекст шортовый

Это **статично**. А набор позиции (через модель m5m1, слом структуры на m3, m15m1 и т.д.) — уже **динамика** с вариациями по ситуации.

Важно: любой гибрид должен быть оттестирован и иметь внутреннюю логику.
