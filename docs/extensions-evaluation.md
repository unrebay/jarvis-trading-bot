## Детальная оценка

### ✅ GitHub MCP — НУЖЕН СЕЙЧАС

**Что делает:** Прямое управление GitHub-репозиторием из Claude — commit, push, read files, PR, issues.

**Зачем нам:** Сейчас Claude на iMac не может push в GitHub (proxy 403). С GitHub MCP — сможет напрямую коммитить файлы без терминала. Критично для workflow с двумя машинами.

**Установить:** Claude Desktop → Settings → Extensions → GitHub MCP

---

### ✅ Supabase MCP — АКТИВЕН

**Что делает:** Управление Supabase проектом прямо из Claude — SQL, миграции, логи, таблицы.

**Статус:** Уже подключён. Уже использован для создания таблиц и функций.

---

### ✅ Claude Skill Creator — АКТИВЕН

**Что делает:** Создание и улучшение скиллов (промпт-инструкций) для Claude.

**Статус:** Доступен. Использовать для улучшения knowledge-structurer и trading-mentor.

---

### ✅ context7 — НУЖЕН СЕЙЧАС

**Что делает:** Подтягивает актуальную документацию библиотек прямо в контекст Claude. Вместо устаревших знаний из обучения — живые доки.

**Зачем нам:** При написании кода для python-telegram-bot, anthropic SDK, supabase-py, asyncio — Claude будет видеть актуальные API, а не версии 2-летней давности.

**Установить:** Claude Desktop → Extensions → context7 (или через MCP registry)

---

### ⏳ n8n-mcp — ПОЗЖЕ (Этап 2)

**Что делает:** Автоматизация через n8n — Claude может триггерить workflow (webhook, cron, pipelines).

**Зачем нам:** Для автоматизации ingestion pipeline: новый файл в папке → автоматически запускает knowledge-structurer. Или ночной rebuild knowledge_base.

**Когда:** После того как основной пайплайн заработает вручную.

**Требует:** n8n инстанс (на VPS или cloud).

---

### ⏳ Obsidian MCP — ПОЗЖЕ (опционально)

**Что делает:** Интеграция с Obsidian — чтение/запись заметок, vault.

**Зачем нам:** Если захочешь вести торговый дневник в Obsidian и синхронизировать с KB.

**Когда:** Только если сам используешь Obsidian. Не критично.

---

### ⏳ GSD (Get Shit Done) — ПОЗЖЕ

**Что делает:** Система управления задачами для Claude — структурированные планы, приоритеты, прогресс.

**Зачем нам:** Может улучшить управление задачами между iMac/MacBook сессиями.

**Ссылка:** github.com/glittercowboy/get-shit-done

**Когда:** Когда проект масштабируется и задач станет много.

---

### 📚 Awesome Claude Code — СПРАВОЧНИК

**Что делает:** Коллекция промптов, конфигов, лучших практик для Claude Code.

**Зачем нам:** Полезен как справочник для улучшения промптов. Не MCP-расширение, а репозиторий с идеями.

**Рекомендация:** Периодически просматривать для вдохновения.

---

### ❌ Claude Mem — НЕТ

**Что делает:** Persistent memory для Claude между сессиями.

**Почему нет:** У нас уже есть встроенный auto-memory (файловая система в `.auto-memory/`). Добавлять ещё один слой памяти — избыточно и может создать конфликты.

---

### ❌ UI/UX Pro Max Skill — НЕТ

**Что делает:** Скилл для UI/UX дизайна, Figma, макеты.

**Почему нет:** Не относится к торговому боту. Нет frontend-задач в нашем проекте.

---

### ❌ Security Guidance — НЕТ

**Что делает:** Проверка безопасности кода, уязвимостей.

**Почему нет:** Не критично для текущей фазы. Добавить потом перед production-деплоем если нужно.

---

### ❌ Dev Browser — НЕТ

**Что делает:** Браузерная автоматизация для разработки.

**Почему нет:** У нас уже есть Claude in Chrome MCP, который покрывает все задачи браузерного скрапинга.

---

### ❌ Notebook LM Python — НЕТ

**Что делает:** Интеграция с Google NotebookLM или Jupyter notebooks.

**Почему нет:** Нет кейса. Jupyter заменяем обычным Python. NotebookLM не интегрируется с нашим стеком.

---

### ❓ Superpowers — УТОЧНИТЬ

**Что делает:** Зависит от конкретного пакета. Нужна ссылка.

**Рекомендация:** Пришли ссылку — оценю.

---

## Что установить прямо сейчас

1. **GitHub MCP** — в Claude Desktop → Settings → Extensions
2. **context7** — в Claude Desktop → Extensions или `claude mcp add context7`

Остальное — когда закончим фундамент.
