# Order Flow Validation & Invalidation

## Основные правила работы с контекстом

Контекст определяется при помощи потока приказов (OF). Поток приказов начинает формирование после двух подтверждений.

**Подтверждение** — манипуляция с ликвидностью или ценовым дисбалансом, сопровождающаяся реакцией и обновлением текущего максимума/минимума.

## Правило одного подтверждения

При достижении первой возможной отметки — ожидаем одно подтверждение для продолжения тренда. Одно подтверждение говорит об актуальности потока приказов.

## Правило двух подтверждений

При достижении первой возможной отметки — дождаться двух подтверждений для зарождения нового потока приказов. Новый OF = инвалидация предыдущего.

## Инвалидация при недостижении таргета

Если цена не достигла первого выделенного таргета — новый поток приказов рассматриваем только после третьего подтверждения.

## Обнуление информации

Поток приказов формируется из манипуляций и перекрытия ценовых дисбалансов. Инвалидация текущего OF = обнуление информации. Поток приказов становится неактуальным.
